# プロジェクト概要

## このプロジェクトについて

技術トレンドを複数の情報ソース（GitHub、Reddit、Hacker News 等）から集約・可視化し、開発者が効率的にトレンドを把握・比較・継続観測できる Web アプリケーション。

## 解決する課題

- 技術トレンド情報が複数プラットフォームに分散しており、横断的な比較が困難
- 各プラットフォームを個別に巡回する非効率さ
- 一時点のスナップショットではトレンドの持続性を判断できない
- 生データのノイズが多く、重要なシグナルを見落としやすい

## 提供する価値

- 複数ソースの技術トレンドを一画面で横断的に把握
- 注目度の相対的な比較・一覧表示
- 継続観測による一過性のバズと持続的トレンドの区別
- バックエンドで整形・集約済みのデータによるノイズ低減

## 初期フェーズ

- データソースは GitHub に限定
- GitHub のトレンドリポジトリ、スター数推移、言語別トレンドを取得・表示
- ユーザー認証は初期スコープ外（GitHub API トークンはバックエンド管理）

## 将来の拡張方向

- Reddit、Hacker News のデータソース追加
- 複数ソース間のクロスリファレンス
- トレンドの時系列推移可視化
- 通知・アラート機能
- ユーザーごとのウォッチリスト・カスタマイズ
