# ドメイン設計

## コアドメイン

このプロダクトのコアドメインは「技術トレンドの集約と可視化」である。

## エンティティ

### TrendingRepository（トレンドリポジトリ）

技術トレンドの中核となるエンティティ。GitHub のリポジトリ情報をプロダクト固有の形式に整形したもの。

主要属性:
- リポジトリ名
- オーナー名
- 説明文
- 主要言語
- スター数
- フォーク数
- リポジトリ URL
- トレンドスコア（集約・算出した注目度指標）

### Language（プログラミング言語）

フィルタリングや分類に使用する言語情報。

主要属性:
- 言語名
- カラーコード

### TrendPeriod（トレンド期間）

トレンドの集計期間を表す値オブジェクト。

種別:
- daily（日次）
- weekly（週次）

## サービス

### TrendAggregationService（トレンド集約サービス）

- 外部 API から取得した生データをプロダクト固有のエンティティに変換する
- フィルタリング、ソート、集計ロジックを担う
- バックエンドの usecases/ 層に実装する

### ExternalSourceClient（外部ソースクライアント）

- 各データソース固有の API 通信を担う
- バックエンドの clients/ 層に実装する
- データソースごとに独立した実装を持つ（GitHubGraphQLClient 等）

## ドメインルール

- 外部 API のレスポンス型をそのままドメインエンティティとして使用しない
- 外部 API の生データは必ずプロダクト固有のエンティティに変換してからフロントエンドに返す
- トレンドスコアの算出ロジックはバックエンドに閉じ込める
- データソースが増えても、フロントエンドが扱うエンティティの構造は統一する

## 将来の拡張

データソース追加時に想定されるエンティティ:
- RedditPost（Reddit 投稿）
- HackerNewsItem（Hacker News 記事）
- AggregatedTrend（複数ソース横断の集約トレンド）

これらは追加時に具体化する。現時点では GitHub 関連エンティティに集中する。
