# 仕様一覧

機能を実装するための技術的な仕様をここに集約する。

## トレンドリポジトリ取得

### データ取得方式

- バックエンドが GitHub GraphQL API を使用してトレンドデータを取得する
- フロントエンドは GitHub API を直接呼び出さず、バックエンドの REST API のみを利用する
- GitHub API の認証トークンはバックエンドの環境変数で管理する

### レスポンス整形

- GitHub API のレスポンスをそのままフロントエンドに返さない
- バックエンドのユースケース層でプロダクト固有のスキーマに整形・集約する
- 不要なフィールドの除去、命名の統一、データの要約を行う

### API 契約

- バックエンドは Zod OpenAPI Hono でルート定義と同時に OpenAPI スキーマを生成する
- フロントエンドは Orval で OpenAPI 定義から API クライアント・TypeScript 型・Zod スキーマを自動生成する
- 手書きの API 型定義やクライアントコードは作成しない

### フィルタリング

- 言語フィルタ: クエリパラメータで言語を指定し、バックエンドでフィルタリングする
- 期間フィルタ: 日次・週次の期間指定をクエリパラメータで受け取る

### エラーハンドリング

- GitHub API のレート制限超過時は適切なエラーレスポンスを返す
- 外部 API の障害時はフロントエンドに統一されたエラー形式で通知する

## フロントエンド状態管理

- API レスポンスのキャッシュと再取得は Orval 生成クライアント経由で管理する
- フィルタ条件などの UI 状態は features レイヤーの model で管理する
- グローバル状態は最小限に抑え、必要な場合のみ app レイヤーで管理する

## 将来のデータソース追加仕様

- 新しいデータソース追加時はバックエンドにモジュールを追加する
- 各モジュールは同一の層構成（routes / usecases / clients / schemas / types）で実装する
- フロントエンドの API インターフェースへの影響を最小化する
- 外部 API の仕様差分（認証方式、ページネーション、レスポンス形式）はモジュール内に閉じ込める
