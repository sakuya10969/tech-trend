# プロダクト概要

## 目的

技術トレンドを複数の情報ソースから集約・可視化し、開発者が効率的にトレンドを把握・比較・継続観測できる Web アプリケーションを提供する。

## 解決したい課題

- GitHub のトレンドページや Reddit の技術系サブレディットなど、技術トレンド情報は複数のプラットフォームに分散している
- 各プラットフォームを個別に巡回するのは非効率で、横断的な比較が困難
- 一時点のスナップショットだけでは、トレンドの推移や持続性を判断できない
- 生データのままでは情報量が多すぎて、重要なシグナルを見落としやすい

## 提供する価値

- 複数ソースの技術トレンドを一画面で横断的に把握できる
- トレンドの比較・一覧表示により、注目度の相対的な位置づけがわかる
- 継続的な観測を通じて、一過性のバズと持続的なトレンドを区別できる
- バックエンドで整形・集約済みのデータを提供することで、ノイズを減らし本質的な情報に集中できる

## 想定ユースケース

- 開発者が週次・日次で技術トレンドの動向を確認する
- 技術選定の参考として、特定の言語・フレームワーク・ツールの注目度を比較する
- チーム内で技術トレンドの共有・議論の起点として利用する
- 新しい技術やリポジトリの発見に活用する

## 初期フェーズの対象範囲

- データソースは GitHub に限定する
- GitHub のトレンドリポジトリ、スター数推移、言語別トレンドなどを取得・表示する
- バックエンドで GitHub GraphQL API を利用し、データを整形・集約してフロントエンドに提供する
- フロントエンドではトレンドの一覧表示、フィルタリング、基本的な比較機能を実装する
- 認証は GitHub API のトークン管理のみ（バックエンド側で管理、ユーザー認証は初期スコープ外）

## 将来的な拡張方針

- Reddit（r/programming 等の技術系サブレディット）をデータソースとして追加
- Hacker News のフロントページ・トレンド記事の取得
- 複数ソース間のクロスリファレンス（同一技術が複数ソースで言及されている場合の集約）
- トレンドの時系列推移の可視化
- 通知・アラート機能（特定技術の急上昇検知など）
- ユーザーごとのウォッチリスト・カスタマイズ機能

## スコープ外（現時点）

- ユーザー認証・アカウント管理
- データの永続化・履歴蓄積（初期はリアルタイム取得ベース）
- モバイルアプリ対応
- 多言語対応（UI は日本語または英語の単一言語で開始）
